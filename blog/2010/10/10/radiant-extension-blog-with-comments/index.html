Radiant extension: blog-with-comments

Posted by Star on October 10, 2010 | 17 comments

Radiant CMS is the most popular CMS system in the open source community and on the market. As a matter of fact, Ruby language website uses Radiant.

Now, almost all companies and many personal websites have a blog. Radiant CMS can be used as a blog system, but it only can post and display articles and doesn’t provide rights management, The rights management is important when you have multiple bloggers and want to make them work independently. Currently, we haven’t found a solid implementation on the market (on github), so we have created our own and share with you! You can find it @ http://www.imustardsoft.com/blog/2010/09/26/introducing-radiant-blog-extension.

Another important feature of blog is comments. There are already quite a few in github, out of which we think http://github.com/saturnflyer/radiant-comments is probably the best. This extension has many advantages and supports the following features:
1. have sample questions to prevent comment spamming
2. E-mail notification when someone comments on your blog
3. approve and modify the comment content

Now we have fork it and imploded it to our blog extension. so, a new extension of radiant “blog-with-comments” is born http://github.com/imustardsoft/radiant-comments. Relative to the radiant-blog-extension, the blog-with-comments-extension not only has all the features and fully-fledged comments system to it, but incorporates the rights management to make the system more robust.
So it is a fully functional blog extension of radiant. You are welcome to use it and let us know what you think.

Installation

To install Comments, run: rake radiant:extensions:comments:install

Setup

1. In your site layout (or another appropriate place) use the following code to display your comment form:
<r:snippet name=“comments” />
2. To use email notification, you should Delete the following line in config/environment.rb 


      config.frameworks -= [ :action_mailer ]

Or just remove the :action_mailer references

      config.frameworks -= []

3. If you’re using SMTP to send emails, create the following to /config/initializers/action_mailer.rb

    ActionMailer::Base.delivery_method = :smtp
    ActionMailer::Base.raise_delivery_errors = true
    ActionMailer::Base.smtp_settings = {
      :enable_starttls_auto =>  true,
      :address              =>  "smtp.gmail.com",
      :port                 =>  "587",
      :domain               =>  "smtp.gmail.com",
      :authentication       =>  :plain,
      :user_name            =>  "username@gmail.com",
      :password             =>  "password"
    }
